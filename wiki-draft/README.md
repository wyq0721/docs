# 正式 Wiki 初稿

这套目录基于 `/templates/wiki/` 生成，可直接作为一版“可填写的正式 wiki 初稿”使用。

## 建议使用方式

1. 先优先填写：
   - `01-项目总览.md`
   - `03-典型案例库.md`
   - `04-方法论沉淀.md`
2. 再根据已有素材补充时间线、效果、风险与对外表达部分。
3. 填写完成后，可继续按实际项目情况增删章节。

## 推荐目录结构

```text
01-项目总览
02-推进时间线
03-典型案例库
04-方法论沉淀
05-效果与价值证明
06-风险与治理
07-对外表达素材库
08-附录
```

## 文件说明

- `01-项目总览.md`：说明为什么做、做什么、当前整体状态。
- `02-推进时间线.md`：记录从启动到当前阶段的推进过程。
- `03-典型案例库.md`：沉淀可复用案例。
- `04-方法论沉淀.md`：总结场景选择与落地方式。
- `05-效果与价值证明.md`：整理效率、质量、组织价值证据。
- `06-风险与治理.md`：明确边界、审核与治理要求。
- `07-对外表达素材库.md`：为 PPT、汇报、对外介绍准备素材。
- `08-附录.md`：归档工具、模板、FAQ 与资料索引。

## 填写原则

- wiki 写完整，PPT 从 wiki 抽取。
- 优先写事实，再写判断，最后写对外表达。
- 所有对外内容都应基于人工复核后的信息。
